Contexto
O site da DomRealce já está online e funcional. É um projeto HTML, CSS e JavaScript puros (sem frameworks).
Quero aplicar melhorias incrementais de performance, SEO, acessibilidade e conversão, sem reescrever a base do site nem quebrar funcionalidades.
As imagens reais serão fornecidas depois; por enquanto, use placeholders otimizados.

Objetivo
Aprimorar a homepage para ficar mais rápida, clara e otimizada para SEO, mantendo identidade (preto/amarelo) e o layout atual.

Regras (muito importante)

Não remover CSS/JS/HTML existentes; apenas adicionar.

Criar novos ficheiros:

/assets/css/domrealce-enhancements.css

/assets/js/domrealce-enhancements.js

Injetar estes dois ficheiros depois dos ficheiros atuais no <head> (CSS) e antes de </body> (JS).

Comentar todas as inserções com <!-- DOMREALCE: ... -->.

2) Overlay leve no banner (melhor leitura do texto)

Sem alterar o HTML do herói, apenas via CSS (funciona com qualquer section que tenha imagem de fundo):

/* DOMREALCE: overlay no hero sem mexer no HTML */
.dr-hero-overlay{
  position:relative; isolation:isolate;
}
.dr-hero-overlay::after{
  content:""; position:absolute; inset:0;
  background:linear-gradient(to bottom, rgba(0,0,0,.35), rgba(0,0,0,.15));
  z-index:-1; /* abaixo do conteúdo, acima do bg */
}


Como aplicar sem editar markup existente:
No /assets/js/domrealce-enhancements.js, envolva a primeira seção com fundo em uma classe extra:

// DOMREALCE: aplicar overlay no primeiro bloco com imagem de fundo
(function(){
  const sections = document.querySelectorAll('section, header, div');
  for (const el of sections) {
    const bg = getComputedStyle(el).backgroundImage || "";
    if (bg.includes('url(')) { el.classList.add('dr-hero-overlay'); break; }
  }
})();

3) Lazy-load em imagens abaixo da dobra

Sem tocar nas tags existentes, faça lazy onde ainda não houver:

// DOMREALCE: lazy-load para imagens sem loading definido
(function(){
  const imgs = document.querySelectorAll('img:not([loading])');
  imgs.forEach(img => { img.setAttribute('loading','lazy'); });
})();

5) Alt text automático (fallback)

Se alguma imagem não tiver alt, criar um alternativo leve:

// DOMREALCE: alt fallback para acessibilidade
(function(){
  document.querySelectorAll('img:not([alt])').forEach((img,i)=>{
    const src = (img.getAttribute('src')||'').split('/').pop() || `imagem-${i+1}`;
    img.setAttribute('alt', src.replace(/[-_]/g,' ').replace(/\..+$/,'').trim());
  });
})();

7) Redireciono seguro de /cantactos → /contactos (estático)

Se o hosting não permite regras de servidor, use fallback JS apenas nessa rota (não afeta o resto):

8) Micro-otimizações de performance

Pré-carregar a imagem principal do herói (não alterar a tag existente, apenas adicionar no <head>):

<!-- DOMREALCE: preload hero image -->
<link rel="preload" as="image" href="/img/placeholder-hero.webp" imagesrcset="/img/placeholder-hero.webp 1x">


Preconnect para fontes externas (se usar Google Fonts, por ex.):

<!-- DOMREALCE: preconnect fonts -->
<link rel="preconnect" href="https://fonts.googleapis.com" crossorigin>
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
